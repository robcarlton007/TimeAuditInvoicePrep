About the Name

This application is named in honor of Luca Pacioli (c. 1447–1517), a 15th-century Franciscan friar and mathematician widely regarded as the "Father of Accounting." The app icon is the rhombicuboctahedron from Pacioli's De Divina Proportione (1509), drawn by Leonardo da Vinci.

Luca carries that tradition forward: ensuring every hour of work is captured, coded correctly, and accounted for before a single invoice goes out the door.

How to Run an Audit

  1. Open Luca.exe
  2. Select a date range using the preset buttons or the From / To date pickers in the sidebar
  3. Select studios — check/uncheck Asheville, Nashville, Memphis
  4. Click Run Audit — reference data is served from local cache; timesheets are fetched live
  5. When complete, click Open Report to view results in your browser, or Print Report to print

Local Cache

Luca stores a local copy of Ajera reference data at ~/Documents/Luca/ajera_cache.json. This cache is used for employees, projects, phases, activities, and the last 10 weeks of timesheet history. Reference data is considered stale after 24 hours.

Rolling Timesheet Window

Timesheets are stored by payroll week (keyed by Friday date), enabling two sync modes:

ModeTriggerWhat it fetches
Full syncSync Data buttonAll reference data + 10 weeks of timesheets
Partial syncBackground refresh after each auditLast 4 weeks of timesheets only

After every sync, weeks older than 10 weeks are automatically dropped. Partial mode ensures retroactive timesheet edits by employees are always captured without re-fetching the entire history.

Most chat questions are answered instantly from the cache, with no API call needed.

What Gets Flagged

Blocked — Project Entries Without Notes

Any project time entry with no description note is flagged and blocked from invoicing. PTO and overhead entries do not require notes.

Incomplete Days

Any Mon–Fri where an employee logged less than their daily minimum (default 8h / 40h week; exceptions in luca_reference.json). Shows hours logged, shortfall, current-week total, and prior-week total.

Full Overhead / General Days

Any day where 90% or more of an employee's hours were coded to overhead or general with no project work.

Missing Timesheet automated mode

Employee has no entries at all for a prior business day. Triggers a personalized AI-written message to the employee with supervisor CC.

Escalations — Consecutive Missed Days roadmap

Three escalation tiers based on consecutive missed days. Messages are AI-written fresh for each case — not scripted templates. See LUCA.md for tier definitions.

Spelling / Typo Detection roadmap

Notes checked against a standard dictionary plus firm-specific whitelist. Flagged words highlighted in the report.

Ask Luca — AI Chat

The Ask Luca panel sits at the bottom of the app window. Powered by Claude (Anthropic) with automatic fallback to Gemini (Google).

The chat is a persistent, scrollable session — history survives across questions and is auto-saved when the app closes. When you reopen the app, the prior session is restored.

Context LUCA has:

  • Complete audit-period timesheet data (all flags, per-employee log, per-project log)
  • 10 weeks of rolling timesheet history (for trend analysis)
  • Full active employee roster with studio and company assignments
  • Firm reference data: overhead code definitions, per-employee hour targets
  • Business rules, phase definitions, and billing confusion patterns (luca_knowledge.yaml)
  • Action log and learned playbook

Example questions:

  • "Who has the most missing notes this week?"
  • "Is Emily's hours trend improving compared to last month?"
  • "Which projects have incomplete entries that could hold up invoicing?"
  • "What is the ProjectKey for the Greenway Park project?"
  • "List all active projects"

Action System — Fully Autonomous

LUCA detects action opportunities within its responses and executes them immediately — no admin initiation, no approval step. When an action is warranted:

  1. LUCA generates the action inline in its response
  2. Executes it automatically
  3. Confirms in chat: "Action executed — REMINDER / Emily Archer"
  4. Logs to luca_actions.json and updates luca_playbook.json

All actions are logged and auditable. To record an outcome: "Mark the reminder to Emily as resolved."

Automated Daily Mode roadmap

When live, Luca runs automatically every weekday morning at 6:00 AM Eastern via GitHub Actions (cloud-hosted, free tier):

No machine at Carlton Edwards needs to be on. The workflow file is at .github/workflows/luca_daily.yml.

Troubleshooting

Timeout on monthly reports — Batches in groups of 10 with a 90-second timeout. Try again; usually succeeds on retry.

No timesheets found — Ajera keys timesheets to the Friday end-of-week. The app adjusts automatically.

Open Report button disabled — Run an audit first.

Ask Luca says "API key required" — Paste your Anthropic API key in the Credentials section of the sidebar.

Chat session lost after restart — Session is auto-saved on close to luca_session.json. If missing or corrupt, the session starts fresh.

Architecture — LLM for Reasoning Only

All data fetching, aggregation, counting, flag detection, cache management, and lookups are deterministic Python code. The LLM (Claude / Gemini) is only invoked for tasks that require reasoning, narrative generation, or natural-language interpretation — writing daily review messages, generating supervisor reports, and answering freeform chat questions. This ensures reliable, repeatable audit results regardless of LLM availability.
